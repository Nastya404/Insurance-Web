# СтрахГарант — Страховая организация

Web-приложение для управления страховыми полисами с REST API на основе Express.js.

## Технологии

| Категория | Технологии |
|-----------|-----------|
| Backend | Node.js, Express.js |
| Frontend | HTML5, SCSS, JavaScript, jQuery |
| Шаблонизатор | EJS |
| Хранение данных | JSON |

## Функционал

### Часть 1 — Backend (REST API)

| Метод | Маршрут | Описание |
|-------|---------|----------|
| GET | `/items` | Получить список всех полисов |
| GET | `/items/:id` | Получить полис по id |
| POST | `/items` | Добавить новый полис |
| PUT | `/items/:id` | Обновить полис |
| DELETE | `/items/:id` | Удалить полис |

### Часть 2 — Frontend

-  **Главная страница** — описание компании и категории страхования
-  **Страховые продукты** — таблица продуктов с ценами и типами
-  **Управление полисами** — CRUD операции через EJS шаблон
-  **Динамическая таблица** — данные из REST API через AJAX

## Структура проекта

```
lab8/
├── public/
│   ├── data/
│   │   └── products.json     — данные страховых продуктов
│   ├── javascripts/
│   │   └── app.js            — клиентский JavaScript (SPA)
│   └── stylesheets/
│       ├── style.scss        — основные стили (SCSS)
│       ├── style.css         — скомпилированный CSS
│       ├── policies.scss     — стили страницы управления
│       └── policies.css      — скомпилированный CSS
├── views/
│   └── policies.ejs          — EJS шаблон управления полисами
├── db.json                   — база данных (JSON)
├── server.js                 — настройки сервера Express
├── rest.js                   — маршруты REST API
├── store.js                  — работа с файлом данных
└── package.json
```

## Установка и запуск

```bash
# Установить зависимости
npm install

# Запустить сервер
npm start

# Компиляция SCSS (в отдельном терминале)
npm run sass
```

Сервер запустится на `http://localhost:3000`

## Архитектура

- **server.js** — настройки сервера, подключение маршрутов
- **rest.js** — маршруты REST API, рендеринг EJS шаблонов
- **store.js** — чтение, запись, обновление db.json
